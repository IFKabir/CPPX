# Security Policy

## Supported Versions

CPPX currently support the following versions of the `stl_ext` library. If you are using an older version, please update to the latest supported version to ensure you have the latest security patches.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.0   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of our software seriously. If you have found a vulnerability in this project, please **do not** open a public issue or pull request, as this may alert malicious actors before a fix is ready.

Instead, please follow these steps:

1.  **Email**: Send a detailed report to **ifk.ishanfalak@gmail.com**.
2.  **Details**: Please include as much information as possible, including:
    - The type of issue (e.g., buffer overflow, memory leak, injection).
    - Full steps to reproduce the issue.
    - Sample code or a proof-of-concept (POC) if possible.
    - Any relevant platform details (OS, Compiler version).

### What to Expect

- **Response**: We will acknowledge receipt of your report within **48 hours**.
- **Assessment**: We will investigate the issue and determine its impact.
- **Fix**: If confirmed, we will release a patch or update as soon as possible.
- **Disclosure**: We will coordinate the public disclosure of the vulnerability with you once a fix is available.

Thank you for helping keep this project safe!
